Llamar al nodo padre y moverse entre los nodos vecinos

Ya saben qué es el DOM, qué es un nodo y cómo se manipula. Aquí resolvemos una sola duda con demos que funcionan: desde un elemento, ¿cómo llego a su padre, a sus hijos o a sus hermanos? Usa → o la barra inferior para avanzar.

PADRE.parentElement.closest('.card')
.previousElementSiblingelemento actual ← aquí estás.nextElementSibling
HIJOS.children.firstElementChild.querySelectorAll()

Cada flecha es un movimiento entre nodos. En la siguiente diapositiva vemos qué hace cada uno; luego, un ejemplo simple por cada método.

Conceptos ¿Qué hace cada uno?

El mapa de movimientos entre nodos

Partiendo siempre de un elemento, estas propiedades y métodos te llevan a los nodos de alrededor. Esta es la idea de cada uno en una frase.

⬆ Subir (hacia el padre)

.parentElementEl elemento que lo contiene directamente: un solo nivel hacia arriba.
.closest('.x')Sube por los ancestros hasta el primero que coincide con el selector, sin importar cuántos niveles haya.

↔ Saltar de lado (hermanos)

.nextElementSiblingEl elemento hermano que viene justo después, dentro del mismo padre.
.previousElementSiblingEl elemento hermano que viene justo antes.

⬇ Bajar (hacia los hijos)

.childrenTodos los elementos hijos (solo las etiquetas, sin los espacios del HTML).
.firstElementChildEl primer hijo; .lastElementChild es el último.
.querySelector('.x')Busca dentro del elemento el primer descendiente que coincide con el selector.
✓

Regla mental: primero ubícate en "el elemento actual", y luego pregunta: ¿necesito su padre (subir), un hermano (de lado) o un hijo (bajar)? Esa pregunta te dice qué propiedad usar.

01 Subir un nivel

Ocultar una tarjeta con parentElement

Lo más simple: .parentElement te da el elemento que contiene a otro (su padre directo). El botón está dentro de la tarjeta, así que subimos un nivel y la ocultamos. Pulsa «Ocultar».

⬆ boton.parentElement = la tarjeta que lo contiene
tarjeta.html 1 salto hacia arriba

Tarjeta de aviso

Este es un mensaje que puedes cerrar.

Pulsa «Ocultar» y mira qué nodo se selecciona.
<div class="tarjeta">
  <p>Mensaje…</p>
  <button>Ocultar</button>
</div>

boton.addEventListener("click", () => {
  // el padre del botón es la tarjeta
  boton.parentElement.style.display = "none";
});
i

Léelo así: "del botón, dame su padre". Como el botón vive dentro de la tarjeta, boton.parentElement es la tarjeta. Un elemento, un salto hacia arriba.

02 Subir varios niveles

Borrar una tarea con closest

A veces el padre no está justo arriba. Aquí el botón está dentro de un grupo, dentro de la tarea. closest('.tarea') sube hasta el primer ancestro con esa clase, sin que cuentes los niveles. Pulsa la ✕.

⬆ boton.closest('.tarea') = sube hasta encontrarla
tareas.html sube hasta .tarea
  • Comprar pan
  • Llamar al médico
  • Pagar la luz
Pulsa una ✕ para borrar esa tarea.
<li class="tarea">
  <span>Comprar pan</span>
  <span class="acciones">
    <button>✕</button>
  </span>
</li>

// el botón NO es hijo directo de la tarea
boton.closest(".tarea").remove();
!

Compara: aquí boton.parentElement sería solo el <span class="acciones">, no la tarea. closest('.tarea') sube directo hasta el <li> sin importar cuántos niveles haya.

03 Bajar a los hijos

Numerar una lista con children

.children te da los elementos hijos de un contenedor (solo las etiquetas, sin los espacios). Recorremos la lista y a cada ítem le ponemos su número. Pulsa «Numerar».

⬇ lista.children = sus elementos hijos (los <li>)
compras.html recorrer hijos
  • Manzanas
  • Leche
  • Huevos
  • Café
La lista tiene 4 hijos.
<ul>
  <li>Manzanas</li>
  <li>Leche</li>
</ul>

// recorremos cada hijo con su posición
lista.children.forEach((li, i) => {
  li.textContent = (i + 1) + ". " + li.textContent;
});
i

¿Por qué children? Devuelve solo los <li> (4 elementos). Su pariente childNodes incluiría también los espacios entre etiquetas y contaría 9. Lo verás en una diapositiva más adelante.

04 Saltar al de al lado

Mostrar la respuesta con nextElementSibling

.nextElementSibling te da el hermano que viene justo después. La respuesta está escrita después de la pregunta, como su hermano. Pulsa una pregunta para revelarla.

↔ pregunta.nextElementSibling = la respuesta que sigue
faq.html hermano siguiente

Pulsa "Registrarse" y completa tu correo.

Sí, el plan básico no tiene costo.

Pulsa una pregunta.
<button>¿Es gratis?</button>
<p>Sí, el plan básico…</p>

pregunta.addEventListener("click", () => {
  // la respuesta es el hermano siguiente
  const resp = pregunta.nextElementSibling;
  resp.classList.toggle("abierta");
});
!

Usa nextElementSibling, no nextSibling. El segundo cuenta también los espacios y saltos de línea del HTML, y a menudo devuelve un espacio en blanco en lugar del párrafo.

05 Saltar al anterior

Subir un "me gusta" con previousElementSibling

.previousElementSibling te da el hermano que viene justo antes. El contador está antes del botón, así que desde el botón saltamos hacia atrás para sumarle uno. Pulsa 👍.

↔ boton.previousElementSibling = el contador anterior
likes.html hermano anterior
0
Pulsa 👍 y mira cómo sube el número de al lado.
<span>0</span>
<button>👍 Me gusta</button>

boton.addEventListener("click", () => {
  // el contador está justo antes del botón
  const cont = boton.previousElementSibling;
  cont.textContent = Number(cont.textContent) + 1;
});
i

La pareja: nextElementSibling mira hacia adelante, previousElementSibling hacia atrás. Ambos saltan de lado entre elementos que comparten el mismo padre.

06 El primero y el último

Destacar extremos con firstElementChild

Dentro de un contenedor, .firstElementChild es su primer hijo y .lastElementChild el último. Útil para marcar el "más vendido" arriba y la "oferta" abajo. Pulsa «Destacar».

⬇ lista.firstElementChild  ·  lista.lastElementChild
ranking.html primer y último hijo
  • Plan Pro
  • Plan Estándar
  • Plan Básico
Pulsa para marcar el primero y el último.
<ul>
  <li>Plan Pro</li>      // primero
  <li>Plan Estándar</li>
  <li>Plan Básico</li>  // último
</ul>

lista.firstElementChild.classList.add("top");
lista.lastElementChild.classList.add("low");
i

Llevan "Element" por la misma razón que los hermanos: evitan los nodos de texto. Sus parientes firstChild / lastChild podrían caer en un espacio en blanco.

07 Subir y luego bajar

Mostrar el error en su grupo

Combinamos dos movimientos: subimos del input a su grupo con closest, y dentro de ese grupo bajamos a su mensaje de error con querySelector. Así tocas el error correcto. Deja el campo vacío y pulsa «Guardar».

⬆ .closest('.campo')  luego  .querySelector('.error') ⬇
formulario.html sube al grupo, baja al error

Deja el campo vacío y pulsa «Guardar».
<div class="campo">
  <input>
  <p class="error"></p>
</div>

// 1) sube del input a su grupo
const grupo = input.closest(".campo");
// 2) baja a SU mensaje de error
const error = grupo.querySelector(".error");
error.textContent = "Falta tu nombre";
✓

Sube y baja: primero closest para llegar al contenedor, luego querySelector para encontrar el hijo correcto dentro de él. Si buscaras .error en toda la página, tocarías el de otro campo.

★ El error más común

childNodes vs children

Si tu lista de 4 <li> "tiene 9 hijos", estás contando los espacios del HTML. Esta es la diferencia que más confunde.

!

Misma lógica para hermanos: nextSibling puede caer en un nodo de texto. Usa siempre la versión «Element»: children, nextElementSibling, firstElementChild.

✓

Regla rápida: si vas a recorrer etiquetas o hacer .classList sobre cada elemento, casi siempre quieres children, no childNodes.

★ Para llevar al proyecto

Buenas prácticas para recorrer nodos

Hazlo así

  • Dale a cada botón su propio clic y muévete relativo a él: el mismo patrón sirve para 1 o 100 tarjetas iguales.
  • Empieza desde el elemento donde ocurre la acción (el botón, el input) y desde ahí sube, baja o salta de lado.
  • Para subir, di QUÉ buscas con closest(selector) en vez de contar saltos.
  • Para bajar, acota al contenedor: cont.querySelector(".err"), no global.
  • Verifica: closest() y los hermanos pueden dar null → if(!nodo) return;

Evita esto

  • La escalera parentElement.parentElement cuando dependes de niveles fijos: cualquier wrapper nuevo la rompe en silencio.
  • childNodes para recorrer etiquetas — incluye espacios. Usa children.
  • nextSibling / firstChild cuando quieres un elemento — usa las versiones «Element».
  • querySelector global para encontrar el hijo de "este" grupo — acótalo al contenedor.
✓

Regla de oro: salvo que el padre sea directo y simple, prefiere closest() — expresa la intención y resiste cambios.
