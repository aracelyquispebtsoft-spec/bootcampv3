Problema 1

Hoisting silencioso

¿Qué pasa si uso una variable antes de declararla? Con var no falla: devuelve undefined sin avisar. Con let obtienes un error claro e inmediato.

❌ con var
console.log(usuario);   // ???
var usuario = "Ana";
console.log(usuario);   // "Ana"
✅ con let
console.log(cliente);   // ???
let cliente = "Beto";
node 1-hoisting.js
› pulsa “Ejecutar var”…
Por qué importa: el undefined de var no avisa y revienta más tarde, lejos del origen. let vive en una Temporal Dead Zone: no existe hasta su línea, así que falla justo donde está el error.
Problema 2

Scope de bloque

var solo respeta el scope de función: ignora cualquier bloque { } (if, for, while), así que la variable “se escapa”. let vive solo dentro del bloque.

❌ con var
if (true) {
  var mensaje = "dentro del if";
}
console.log(mensaje);  // ???
✅ con let
if (true) {
  let mensaje = "dentro del if";
}
console.log(mensaje);  // ???
node 2-scope-bloque.js
› pulsa “Ejecutar var”…
Por qué importa: con var una variable temporal de un if/for contamina toda la función y puede pisar otras. let la mantiene encerrada donde la necesitas.
Problema 3 · ⭐ el clásico de entrevistas

Loops + callbacks asíncronos

Con var todos los setTimeout comparten la misma i y, cuando se ejecutan, el loop ya terminó: leen el valor final. Con let cada vuelta tiene su propia copia.

❌ con var → 3, 3, 3
for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log("i =", i);
  }, 10);
}
✅ con let → 0, 1, 2
for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log("i =", i);
  }, 10);
}
node 3-loop-async.js
› pulsa “Ejecutar var”…
Por qué importa: cuando el callback (setTimeout, click, fetch) corre, el loop ya acabó e i vale 3. let “congela” el valor de cada vuelta. Es el mayor «¡ah!» de la charla.
Problema 4

Re-declaración sin error

var permite declarar la misma variable dos veces y pisa el valor sin avisar. let lo detecta como error de sintaxis.

❌ con var
var total = 100;
// ...300 líneas después...
var total = 0;        // sin aviso
console.log(total);  // ???
✅ con let
let total = 100;
let total = 0;
console.log(total);
node 4-redeclaracion.js
› pulsa “Ejecutar var”…
Por qué importa: en archivos grandes o al unir código, redeclarar con var pisa valores en silencio. let te avisa del choque de nombres antes de ejecutar.
Extra · El concepto detrás de todo

Los 3 tipos de scope

Una variable solo existe dentro de su ámbito. Regla de oro: un scope interno ve hacia afuera, pero no al revés (cajas dentro de cajas).

🌍 global · 🧩 función · 📦 bloque
let pais = "Perú";          // 🌍 global

function mostrar() {
  let ciudad = "Lima";       // 🧩 función
  if (true) {
    let distrito = "Miraflores"; // 📦 bloque
  }
}
mostrar();
console.log(ciudad); // fuera de la función → ???
